
# Git 브랜치 네이밍 가이드

## Git 버전 관리 설명

### 기능 + 플랫폼 기반 명명 + 연구 분류
- 경량화 이나 실험은 바로 `research/` 또는 `experiment/` 접니에로 명확히 구분.
- 브랜치 이름만 봐도 모두가 목적 표기가 가능하게.

### ① `model/` 모델 개발 브랜치
> 플랫폼/모델 기반

```
model/gpt
model/ollama
model/langchain
```

### ② `feature/` 기능 개발 브랜치
> 서버, UI, similarity 등

```
feature/server-similarity
feature/server-ui
```

### ③ `research/` 연구/시험 브랜치
> 경량화, 튜닝 등 모델 연구 환목

```
research/model-quantization
research/sllm-lightweighting
```

### ④ 기타
- `fix/` : 버그 수정
- `refactor/` : 노드 구조 수정
- `release/` : 마지막 버전 테그 및 내부 조정

```
fix/server-similarity-bug
refactor/server-utils
release/v1.2.0
```

---

## 세부 개정 노드 (author 기반)
> 자신의 작업 노드는 `/이름-기능`으로 후장.

```bash
model/ollama/mg-ollama-base
research/model-quantization/mg-ollama-quantization
```

| 타입 | 네이버 규칙 예시 |
|--------|-----------------------|
| 기능 개발용 | `model/<module>/<author>-<feature>` |
| 연구/시험용 | `research/<domain>/<author>-<experiment>` |
| 버그 수정용 | `fix/<module>/<author>-<issue-desc>` |
| 마지막 매직 | `release/<version>` |

## 큰 정리
| 부분 | 피해약간 | 권장 형식 |
|--------|-------------|---------------|
| 대문자 포함 | GitHub, CI/CD 오류 위해 제외 | `lower-case + hyphen(-)` |
| 범위 명확 | `/` 구분자 사용 권장 | `model/ollama/...` |
| 노드 포함 | 직관자, 기능 명시 | `mg-ollama-base` |
